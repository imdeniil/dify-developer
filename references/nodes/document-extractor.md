# Document Extractor Node — Верифицированный справочник

> Протестировано на Dify 1.14.2 self-hosted, 2026-06-18.
> Все примеры — из реальных тестов, не из документации.

## Что это

Нода для автоматического извлечения текстового содержимого из загруженных файлов. Поддерживает как одиночные файлы (`FileSegment`), так и массивы файлов (`ArrayFileSegment`).

## Базовая структура (DSL)

```json
{
  "id": "doc_extractor_node_id",
  "type": "custom",
  "data": {
    "title": "DocExtractor",
    "type": "document-extractor",
    "variable_selector": ["start", "doc"]
  },
  "position": { "x": 400, "y": 200 },
  "sourcePosition": "right",
  "targetPosition": "left",
  "width": 242,
  "height": 89
}
```

## Поддерживаемые форматы и механизмы парсинга

Парсинг выполняется либо локально внутри контейнера `api`/`worker`, либо через внешний сервис Unstructured API:

### 1. Локальный парсинг (не требует внешних сервисов)

* **Plain Text & Code** (`.txt`, `.md`, `.mdx`, `.html`, `.xml`, `.py`, `.js`, `.ts`, `.sql`, `.conf`, `.toml`, `.env`, `.log` и др.):
  Автоматически определяет кодировку с помощью библиотеки `charset-normalizer`. Декодирует текст, игнорируя поврежденные символы.
* **JSON** (`.json`):
  Парсит и форматирует JSON с отступами в 2 пробела для улучшения читаемости.
* **YAML** (`.yaml`, `.yml`):
  Парсит и форматирует YAML с сохранением юникода.
* **PDF** (`.pdf`):
  Парсит страницы файла локально с помощью `pypdfium2`.
* **DOCX** (`.docx`):
  Парсит текстовые блоки и таблицы локально с помощью `python-docx` (библиотека `docx`).
* **CSV** (`.csv`):
  Парсит табличные данные через встроенный модуль `csv` языка Python.
* **Excel** (`.xls`, `.xlsx`):
  Парсит листы и таблицы через `pandas` и `openpyxl`.
* **Subtitles** (`.vtt`):
  Парсит субтитры через `webvtt`. Автоматически объединяет подряд идущие фразы одного и того же спикера в формат `Speaker "text"`.
* **Java Properties** (`.properties`):
  Парсит пары ключ-значение, сохраняя комментарии и пустые строки.
* **Emails** (`.eml`, `.msg`):
  Парсит заголовки и тело писем локально с помощью Python email-библиотек.

### 2. Внешний парсинг (требует настройки Unstructured API)

Форматы **`.doc`**, **`.ppt`**, **`.pptx`** и **`.epub`** не парсятся локально. Для их обработки Dify отправляет запросы к Unstructured API:
* Требуются настройки в `.env`: `UNSTRUCTURED_API_URL` и `UNSTRUCTURED_API_KEY`.
* Если эти переменные не заданы, попытка прочесть такой файл вызовет ошибку `TextExtractionError` (например, `Unstructured API URL is not configured for DOC file processing`).

## Выходные переменные (Outputs)

Нода возвращает одну переменную:

| Переменная | Описание | Тип |
|------------|----------|-----|
| `text` | Извлеченный из файла(ов) текст. Если вход — массив файлов, возвращается `array[string]` с текстом для каждого файла. | `string` / `array[string]` |
