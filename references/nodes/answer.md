# Answer Node — Верифицированный справочник

> Протестировано на Dify 1.14.2 self-hosted, 2026-06-18.

## Что это

Нода ответа (`answer`) используется исключительно в **Chatflow** (приложениях типа Chatbot с продвинутым workflow-режимом). Она отвечает за немедленную отправку текстового ответа (включая стриминг) и ассоциированных файлов пользователю в чат-сессию.

## Базовая структура (DSL)

```json
{
  "id": "answer_node_id",
  "type": "custom",
  "data": {
    "title": "Answer",
    "type": "answer",
    "answer": "Привет! Вот результат работы модели:\n\n{{#llm_node.text#}}",
    "variables": []
  },
  "position": { "x": 720, "y": 282 },
  "positionAbsolute": { "x": 720, "y": 282 },
  "sourcePosition": "right",
  "targetPosition": "left",
  "width": 242,
  "height": 89
}
```

## Параметры ноды

- `answer` (string) — шаблон ответа в формате Markdown. Поддерживает переменные через синтаксис `{{#node_id.variable_name#}}`.
- `variables` (array) — массив для ручного переопределения переменных (в большинстве случаев Dify автоматически парсит шаблон и извлекает зависимости, поэтому этот массив может быть пустым `[]`).

## Outputs

Нода `answer` возвращает:
- `answer` (string) — отрендеренный текст ответа (после подстановки всех шаблонов).
- `files` (array) — плоский список файлов, извлеченных из переданных в шаблоне объектов (например, изображений или документов, сгенерированных предыдущими нодами).

Доступ снаружи:
```json
["answer_node_id", "answer"]
["answer_node_id", "files"]
```

---

## ⚠️ Критические ограничения и особенности поведения

1. **Доступность только в Chatflow (Advanced Chat)**:
   Нода `answer` **не поддерживается** в обычных workflow (Text Generator). Попытка выполнить ее в обычном workflow завершится пропуском или ошибкой, так как обычный workflow завершается строго нодой `end` и не имеет концепции потоковой передачи ответов в чат.
   
2. **Стриминг (Streaming)**:
   Благодаря `execution_type = NodeExecutionType.RESPONSE` под капотом, Dify начинает отправлять содержимое ноды `answer` пользователю сразу же, как только завершается генерация данных в ней, не дожидаясь полного завершения всего workflow (если после ноды `answer` идут другие фоновые ноды).

3. **Автоматическое извлечение файлов**:
   Если в шаблоне `answer` используется переменная-файл (или массив файлов, например от ноды генерации изображений), Dify автоматически распознает класс `FileSegment` / `ArrayFileSegment`, извлечет файлы и прикрепит их к сообщению в чате как вложения.
