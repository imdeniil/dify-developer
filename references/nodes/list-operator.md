# List Operator Node — Верифицированный справочник

> Протестировано на Dify 1.14.2 self-hosted, 2026-06-18.
> Все примеры — из реальных тестов, не из документации.

## Что это

Нода для обработки массивов (фильтрация, сортировка, ограничение длины, извлечение элемента по индексу). Работает с массивами строк, чисел, булевых значений и файлов.

## Базовая структура (DSL)

```json
{
  "id": "list_operator_node_id",
  "type": "custom",
  "data": {
    "title": "Filter Fruits",
    "type": "list-operator",
    "variable": ["source_node_id", "fruits"],
    "filter_by": {
      "enabled": true,
      "conditions": [
        {
          "key": "",
          "comparison_operator": "contains",
          "value": "a"
        }
      ]
    },
    "order_by": {
      "enabled": true,
      "key": "",
      "value": "desc"
    },
    "limit": {
      "enabled": true,
      "size": 3
    },
    "extract_by": {
      "enabled": true,
      "serial": "2"
    }
  },
  "position": { "x": 720, "y": 200 },
  "sourcePosition": "right",
  "targetPosition": "left",
  "width": 242,
  "height": 89
}
```

## Порядок выполнения операций

Все операции внутри одной ноды выполняются в строго фиксированном порядке (независимо от того, как они настроены в UI):

1. **Фильтрация (`filter_by`)** — отсеивает элементы по условиям.
2. **Извлечение по индексу (`extract_by`)** — извлекает один элемент по его 1-based порядковому номеру. Результат оборачивается в список из одного элемента.
3. **Сортировка (`order_by`)** — упорядочивает оставшиеся элементы.
4. **Лимит (`limit`)** — ограничивает максимальное число возвращаемых элементов.

## Поддерживаемые типы и операторы фильтрации

### 1. Массивы строк (`ArrayStringSegment`)
Поддерживает операторы:
* `contains` (содержит) / `not contains` (не содержит)
* `start with` (начинается с) / `end with` (заканчивается на)
* `is` (равно) / `is not` (не равно)
* `in` (входит в список) / `not in` (не входит в список)
* `empty` (пусто) / `not empty` (не пусто)

### 2. Массивы чисел (`ArrayNumberSegment`)
Поддерживает операторы сравнения:
* `=` (равно), `≠` (не равно)
* `<` (меньше), `≤` (меньше или равно)
* `>` (больше), `≥` (больше или равно)

### 3. Массивы булевых значений (`ArrayBooleanSegment`)
Поддерживает только операторы:
* `is` (равно)
* `is not` (не равно)

### 4. Массивы файлов (`ArrayFileSegment`)
Позволяет фильтровать по свойствам файла (поле `key` в условии):
* По текстовым свойствам (`name`, `extension`, `mime_type`, `url`, `related_id`): поддерживаются все строковые операторы.
* По системным свойствам (`type`, `transfer_method`): поддерживаются операторы `in` / `not in`.
* По размеру (`size`): поддерживаются числовые операторы сравнения.

## Извлечение по индексу (`extract_by`)

* Индекс (`serial`) задается как 1-based строка (например, `"1"`, `"2"`). Может содержать шаблонную переменную (например, `{{#start.index#}}`).
* **Важно:** индекс должен быть строго $\ge 1$. Индекс $< 1$ вызывает ошибку `ValueError`.
* Если индекс превышает текущую длину отфильтрованного массива, нода падает с ошибкой `InvalidKeyError` (например, `Invalid serial index: must be <= 3, got 5`).
* После успешного извлечения результат приводится к массиву, содержащему ровно один найденный элемент.

## Выходные переменные (Outputs)

Любая нода List Operator возвращает три переменные:

| Переменная | Описание | Тип |
|------------|----------|-----|
| `result` | Обработанный (отфильтрованный/отсортированный) массив | `ArraySegment` (соответствует типу входа) |
| `first_record` | Первый элемент итогового массива (или `None`, если массив пуст) | Совпадает с типом элемента |
| `last_record` | Последний элемент итогового массива (или `None`, если массив пуст) | Совпадает с типом элемента |
