# Human Input Node — Верифицированный справочник

> Протестировано на Dify 1.14.2 self-hosted, 2026-06-18.
> Все примеры — из реальных тестов, не из документации.

## Что это

Нода для приостановки (паузы) выполнения workflow с ожиданием действия человека (Human-in-the-Loop). Предоставляет интерактивную форму в UI или по API, собирает входные данные от пользователя и направляет дальнейший поток выполнения по выбранной кнопке-ветке.

## Базовая структура (DSL)

```json
{
  "id": "human_in_node_id",
  "type": "custom",
  "data": {
    "title": "Awaiting Human",
    "type": "human-input",
    "form_content": "Пожалуйста, подтвердите операцию. Текущее значение: {{#code_node.result#}}\nКомментарий: {{#$output.comment#}}",
    "inputs": [
      {
        "output_variable_name": "comment",
        "type": "paragraph",
        "default": { "type": "constant", "value": "Approved by default" }
      }
    ],
    "user_actions": [
      { "id": "approve", "title": "Approve request", "button_style": "primary" },
      { "id": "reject", "title": "Reject request", "button_style": "accent" }
    ],
    "timeout": 1,
    "timeout_unit": "hour"
  },
  "position": { "x": 400, "y": 200 }
}
```

## Конфигурация формы и полей

### 1. Форма (`form_content`)
Текст формы пишется в формате Markdown. Поддерживает подстановку:
* Обычных переменных workflow: `{{#node_id.var_name#}}`.
* Динамических полей ввода формы: `{{#$output.input_variable_name#}}`. При рендеринге до отправки они остаются плейсхолдерами, а после отправки заменяются на введенный пользователем текст (или на `[file]` / `[X files]` для вложений).

### 2. Поля ввода (`inputs`)
Поддерживаемые типы полей:
* `paragraph` — текстовое многострочное поле (в коде возвращает `string`).
* `select` — выпадающий список выбора.
* `file` — одиночный файл (в коде возвращает `FileSegment`).
* `file-list` — список файлов (в коде возвращает `ArrayFileSegment`).

### 3. Действия (`user_actions`)
Каждое действие превращается в интерактивную кнопку на форме. Содержит `id`, `title` (текст на кнопке) и `button_style` (`primary`, `default`, `accent`, `ghost`).
* **Важно:** `id` действий служат именами исходящих портов (source handles) для ветвления графа.

## Механизм работы HITL (Пауза и Резюме)

Когда выполнение workflow доходит до ноды Human Input:

1. **Пауза:** Нода видит, что форма еще не создана/не заполнена, создает запись формы в таблице `human_input_forms` со статусом `waiting` и выбрасывает системное событие `PauseRequestedEvent`. Поток выполнения текущей задачи Celery приостанавливается.
2. **Получение токена:** Для продолжения работы требуется получить одноразовый `form_token` формы. На консоли Dify это делается через запрос:
   ```bash
   GET /console/api/workflow/<workflow_run_id>/pause-details
   ```
   В ответе вернется URL вида `.../form/human_input/<form_token>` (если `APP_WEB_URL` в `.env` пуст, `backstage_input_url` будет равен `None`, но токен всегда можно найти напрямую в БД в таблице `human_input_form_recipients` по `form_id`).
3. **Отправка формы:** Внешний пользователь или интерфейс отправляет заполненную форму обратно:
   ```bash
   POST /console/api/form/human_input/<form_token>
   {
       "action": "approve",                 # ID нажатой кнопки
       "inputs": {
           "comment": "My custom comment"  # значения полей ввода
       }
   }
   ```
4. **Резюме:** Dify обновляет статус формы на `submitted`, восстанавливает поток выполнения и запускает ту же ноду повторно. Нода считывает введенные данные из БД и завершается с успехом, перенаправляя выполнение по ребру, соответствующему кнопке (например, по порту `approve`).
5. **Таймаут:** Если форма не заполнена до истечения `timeout` (задается в часах или днях), статус формы меняется на `timeout`, а выполнение возобновляется по порту таймаута (`__timeout`).

## Выходные переменные (Outputs)

Нода возвращает все пользовательские поля ввода (например, `comment`) плюс три системных переменных:

| Переменная | Описание | Тип |
|------------|----------|-----|
| `__action_id` | ID выбранной пользователем кнопки (например, `"approve"`) или `""` при таймауте. | `string` |
| `__action_value` | Текстовый заголовок выбранной кнопки (например, `"Approve request"`). | `string` |
| `__rendered_content` | Финальный Markdown-текст формы, где плейсхолдеры входов заменены на введенные данные. | `string` |
| `[input_name]` | Значения полей ввода, заполненных пользователем. | Зависит от типа входа |

## Ветвление (Routing)

Исходящие ребра от ноды Human Input подключаются к конкретным ручкам (source handles), имена которых совпадают с `id` кнопок (например, `sourceHandle: "approve"`, `sourceHandle: "reject"`). Особое ребро таймаута подключается к ручке `__timeout`.
