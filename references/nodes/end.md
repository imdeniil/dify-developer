# End Node — Верифицированный справочник

> Протестировано на Dify 1.14.2 self-hosted, 2026-06-18.

## Что это

Конечная нода (`end`) для обычных workflow. Она завершает выполнение текущей ветки workflow и определяет структуру и значения финального ответа, который возвращается пользователю или вызывающему API.

## Базовая структура (DSL)

```json
{
  "id": "end_node_id",
  "type": "custom",
  "data": {
    "title": "End",
    "type": "end",
    "outputs": [
      {
        "variable": "final_text",
        "value_selector": ["llm_node_id", "text"]
      },
      {
        "variable": "status_code",
        "value_selector": ["code_node_id", "status"]
      }
    ]
  },
  "position": { "x": 720, "y": 282 },
  "positionAbsolute": { "x": 720, "y": 282 },
  "sourcePosition": "right",
  "targetPosition": "left",
  "width": 242,
  "height": 89
}
```

## Конфигурация выходов (Outputs Mapping)

Каждый элемент в массиве `outputs` описывает одно выходное поле финального ответа:

| Поле в элементе | Описание | Пример |
|-----------------|----------|--------|
| `variable` | Имя переменной в результирующем JSON-ответе | `"final_text"` |
| `value_selector` | Путь (массив) к переменной-источнику из предыдущих нод | `["llm_node_id", "text"]` |

## Поведение в workflow

1. **Завершение выполнения**: При достижении ноды `end` выполнение данной ветки workflow останавливается.
2. **Формирование ответа**: Собранные значения из `value_selector` упаковываются в JSON-объект вида:
   ```json
   {
     "final_text": "Значение из LLM ноды...",
     "status_code": 200
   }
   ```
3. **Множественные конечные ноды**: В сложных workflow с условным ветвлением (`if-else`) может быть несколько нод `end`. Выполненной считается та, до которой дошел поток управления.
4. **Использование в итерациях (Iteration)**: Внутри нод циклов `iteration` нода `end` используется для определения того, какое значение возвращается из одной итерации цикла в результирующий массив.
