# Неверифицированные возможности платформы Dify (v1.14.2)

Этот документ содержит список возможностей, сущностей и комбинаций связей, которые задокументированы в нашем справочнике на основе исходного кода и схем Dify, но **не проходили «живое» тестирование** в нашей среде.

---

## 1. Остающиеся неверифицированными возможности

### А. Трассировка внешних запусков (OTel / Langfuse / LangSmith)
*   **Что описано**: Настройка и интеграция внешних провайдеров мониторинга через эндпоинты `/trace-config` и `/trace` (описано в [12-monitoring-runs.md](12-monitoring-runs.md)).
*   **Статус**: Не тестировалась живая отправка спанов в инстансы Langfuse/LangSmith.

### Б. Plugin-триггер (`trigger-plugin`)
*   **Что описано**: Запуск workflow по событиям установленных плагинов (описано в [nodes/trigger.md](nodes/trigger.md)).
*   **Статус**: Не тестировался ввиду отсутствия активных плагинов-триггеров в нашей текущей инсталляции Dify.

### В. Параллельное выполнение с сайд-эффектами
*   **Что за сценарий**: Запуск независимых веток воркеров Celery, совершающих запись во внешние базы данных или отправку внешних API (сайд-эффекты) одновременно.
*   **Ожидаемое поведение**: Ветки выполняются асинхронно и параллельно.
*   **Статус**: Не тестировалось влияние высокой конкурентности на внешние системы и возможное проявление состояний гонки (race conditions).

---

## 2. Верифицировано в ходе тестов (18 июня 2026)

Следующие возможности были успешно протестированы на локальном инстансе Dify v1.14.2 с использованием автоматически создаваемых через Console API тестовых приложений:

### А. Group Mode в Variable Aggregator
*   **Результат**: Настройка `advanced_settings.group_enabled: true` и разделение переменных по `groups` в [nodes/variable-aggregator.md](nodes/variable-aggregator.md) полностью работоспособны.
*   **Специфика выходов**: Нода возвращает словарь, где ключами являются `group_name`, а значениями — объекты вида `{"output": <value>}`. В DSL-структуре при этом базовый массив `variables` должен быть пустым `[]`.
*   **Как верифицировано**: Скриптом [verify_aggregator_group.py](file:///home/keemor/.gemini/antigravity-cli/brain/dbb88615-c8d3-48f9-80d2-f56a01b767d2/scratch/verify_aggregator_group.py).

### Б. Интерактивная пауза и возобновление в Human Input (HITL) & Edge Branching
*   **Результат**: Выполнение успешно приостанавливается на ноде `human-input`. Генерируется `form_token`, который можно извлечь из SSE-потока. После отправки решения на `POST /console/api/form/human_input/<form_token>` выполнение возобновляется.
*   **Маршрутизация**: Проверено ветвление по handles: при отправке `action: approve` активируется только ребро с `sourceHandle: approve`, а альтернативные ветки переводятся в статус `skipped`.
*   **Как верифицировано**: Скриптом [verify_human_input.py](file:///home/keemor/.gemini/antigravity-cli/brain/dbb88615-c8d3-48f9-80d2-f56a01b767d2/scratch/verify_human_input.py).

### В. Мульти-входящие связи в стандартные ноды (Direct standard node join)
*   **Результат**: Подключение нескольких исходящих ребер напрямую к одной ноде (без Variable Aggregator) работает корректно. Пропущенная ветка (`Code Rejected`) была проигнорирована, и нода `End` успешно выполнилась сразу после завершения активной ветки (`Code Approved`) без зависаний.
*   **Как верифицировано**: Интегрировано в тест-кейс скрипта [verify_human_input.py](file:///home/keemor/.gemini/antigravity-cli/brain/dbb88615-c8d3-48f9-80d2-f56a01b767d2/scratch/verify_human_input.py).

### Г. Webhook-триггер (`trigger-webhook`)
*   **Результат**: Автоматический запуск опубликованного workflow через POST-запрос к сгенерированному Dify URL вида `/triggers/webhook/<webhook_id>` работает корректно в фоновом режиме Celery.
*   **Передача данных**: Данные из JSON-тела вебхука успешно маппятся в переменные триггера и передаются по цепочке в последующие ноды.
*   **Как верифицировано**: Скриптом [verify_webhook.py](file:///home/keemor/.gemini/antigravity-cli/brain/dbb88615-c8d3-48f9-80d2-f56a01b767d2/scratch/verify_webhook.py).
